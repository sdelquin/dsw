## Resultados de aprendizaje / Criterios de evaluación

Los resultados de aprendizaje identifican lo que el alumnado sabrá, comprenderá y podrá hacer al final de la formación recibida en el módulo profesional.

Cada resultado de aprendizaje viene definido por una serie de **criterios de evaluación**.

```
~~~graph-easy --as=boxart
[ RA ] -> [ CE ]
~~~
```

---

### RA1: Selecciona las arquitecturas y tecnologías de programación web en entorno servidor, analizando sus capacidades y características propias.

- **Diferencia modelos de ejecución** en servidor y en cliente web.
- Reconoce las **ventajas de la generación dinámica de páginas**.
- Identifica **mecanismos de ejecución de código** en los servidores web.
- Reconoce las funcionalidades de los **servidores de aplicaciones**.
- Identifica lenguajes y **tecnologías de programación web servidor**.
- Verifica **integración de lenguajes de marcas** en programación web.
- Reconoce **herramientas y frameworks de programación web servidor**.

---

### RA2: Escribe sentencias ejecutables por un servidor web reconociendo y aplicando procedimientos de integración del código en lenguajes de marcas.

- Reconoce la generación de código desde **lenguajes de marcas embebidos**.
- Identifica las principales **tecnologías asociadas**.
- Utiliza etiquetas para **inclusión de código en lenguajes de marcas**.
- Reconoce la **sintaxis del lenguaje de programación** a utilizar.
- Escribe **sentencias simples** y comprueba sus efectos en el resultado.
- Utiliza **directivas para modificar el comportamiento predeterminado**.
- Utiliza **distintos tipos de variables y operadores**.
- Identifica los **ámbitos de utilización de las variables**.

---

### RA3: Escribe bloques de sentencias embebidos en lenguajes de marcas, seleccionando y utilizando las estructuras de programación.

- Utiliza **mecanismos de decisión** en la creación de sentencias.
- Utiliza **bucles** y verifica su funcionamiento.
- Utiliza matrices (**arrays**) para almacenar y recuperar datos.
- Crea y utiliza **funciones**.
- Utiliza **formularios web** para interactuar con el usuario.
- Emplea **métodos para recuperar información** desde un formulario.
- Añade **comentarios** al código.

---

### RA4: Desarrolla aplicaciones web embebidas en lenguajes de marcas analizando e incorporando funcionalidades según especificaciones.

- Identifica mecanismos para **mantenimiento de información de cliente**.
- Utiliza mecanismos para mantener el **estado de las aplicaciones web**.
- Utiliza mecanismos para **almacenar información en el cliente web**.
- Identifica mecanismos para la **autentificación de usuarios**.
- Escribe aplicaciones que integren **mecanismos de autentificación**.
- Utiliza herramientas y **entornos para facilitar la programación**.

---

### RA5: Desarrolla aplicaciones web identificando y aplicando mecanismos para separar el código de presentación de la lógica de negocio.

- Identifica las ventajas de **separar lógica de negocio y presentación**.
- Utiliza **mecanismos y frameworks** que facilitan esta separación.
- Utiliza **objetos/controles en el servidor** para generar aspecto visual.
- Utiliza **formularios generados de forma dinámica**.
- Identifica y **aplica parámetros de configuración** de la aplicación web.
- Escribe aplicaciones web con **mantenimiento de estado**.
- Aplica principios y **patrones de diseño de POO**.
- **Prueba y documenta** el código.

---

### RA6: Desarrolla aplicaciones web de acceso a almacenes de datos, aplicando medidas para mantener la seguridad y la integridad de la información.

- Analiza las tecnologías que permiten el **acceso a almacenes de datos**.
- Crea aplicaciones que establezcan **conexiones con bases de datos**.
- **Recupera información** almacenada en bases de datos.
- **Publica** en aplicaciones web la información recuperada.
- Utiliza **conjuntos de datos** para almacenar la información.
- Crea aplicaciones web que permita **actualizar una base de datos**.
- **Prueba** y **documenta** las aplicaciones web.

---

### RA7: Desarrolla servicios web reutilizables y accesibles mediante protocolos web, verificando su funcionamiento.

- Reconoce las **ventajas de utilizar servicios web**.
- Identifica **tecnologías y protocolos** en el consumo de un servicio web.
- **Utiliza estándares** en el desarrollo de servicios web.
- **Programa** un servicio web.
- **Verifica** el funcionamiento del servicio web.
- **Consume** el servicio web.
- **Documenta** el servicio web.

---

### RA8: Genera páginas web dinámicas analizando y utilizando tecnologías y frameworks del servidor web que añadan código al lenguaje de marcas.

- Identifica **diferencias entre código servidor y código cliente**.
- Reonoce las ventajas de **unir ambas tecnologías**.
- Identifica tecnologías y **frameworks** para generar web en servidor.
- Utiliza estas tecnologías para incluir **interacción de usuario**.
- Utiliza estas tecnologías para incluir **verificación de formularios**.
- Utiliza estas tecnologías para incluir **modificación dinámica**.
- **Aplica estas tecnologías** para la programación de aplicaciones web.

---

### RA9: Desarrolla aplicaciones web híbridas seleccionando y utilizando tecnologías, frameworks servidor y repositorios heterogéneos de información.

- Reconoce las ventajas de **reutilizar el código**.
- Identifica **tecnologías y frameworks** para creación de webs híbridas.
- **Crea una aplicación web** que procese repositorios de información.
- **Crea repositorios específicos** a partir de información ya existente.
- **Utiliza librerías y frameworks** para incorporar funcionalidades.
- Programa servicios y aplicaciones web **a partir de código de terceros**.
- Analiza y utiliza librerías de código relacionadas con **Big Data**.
- **Prueba, depura y documenta** las aplicaciones generadas.
